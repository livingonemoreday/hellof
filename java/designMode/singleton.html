Double-Check

如果目标类中存在其他成员，并且是在构建目标单例对象的时候初始化成员属性，由于指令重排序可能存在单例实例化成功了，但是成员还没有初始化完成，在调用单例对象的成员的方法时就可能抛出空指针异常

public class Singleton {

    private B b;
    
    private static Singleton singleton;

    private Singleton() {
        // 构建singleton和构建b有指令重排序的可能？IDEA是有提示将singleton标识为volatile
        this.b = new B();
    }

    public static Singleton getInstance() {
        if (singleton == null) {
            synchronized (A.class) {
                if (singleton == null) {
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }

    public B getB() {
        return b;
    }
}

Holder

public class Singleton {

    private Singleton() {
    }

    public static Singleton getInstance() {
        // 调用Holder时才会初始化Holder，调用Singleton中的其他方法不会初始化Holder
        return Holder.singleton;
    }

    private static class Holder {
        // clinit 线程安全
        private static Singleton singleton = new Singleton();
    }
}

enum是final的，实例是final static的，也是线程安全的，但是没法懒汉加载，因为实例都是static的，会纳入clinit，调用枚举的任何方法都会使得目标单例被加载。

改造懒加载

public class Singleton {

    private Singleton() {
    }
    
    public static Singleton getInstance() {
        return EnumHolder.INSTANCE.getSingleton();
    }

    private enum EnumHolder {
        INSTANCE;
        
        private Singleton singleton;

        EnumHolder() {
            this.singleton = new Singleton();
        }

        private Singleton getSingleton() {
            return singleton;
        }
    }
}
