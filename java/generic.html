由来

一般的类和方法，只能使用具体的类型（基本类型或自定义引用类型等），难以编译出能够作用于多种类型的代码， 比如void doSomething(HashMap map);，就只能传入HashMap类型的参数（包括其子类）方能调用该方法。当然亦可 使用多态机制，将参数类型设置为基类void doSomething(Map map);，but，如果想使其能够应用于某种不确定的类型， 而非一个具体的接口或类，就显得有些强人所难了。so，泛型就应运而生了。

interface X<T> {
    void doSomething(T t);
}

擦除

泛型机制乃1.5引入，需考虑“迁移兼容性”问题，so，有了泛型擦除，即：在泛型代码内部，无法获取任何有关泛型参数类型的信息。 通俗而言，泛型仅仅是对于编译期而言的。

new ArrayList<A>() == new ArrayList<B>()

泛型不能用于显示地引用运行时类型的操作之中，类如转型/instanceof/new表达式，因为所有关于参数的类型信息都丢失了。

HashMap<String, Object> map = new HashMap<>(4);
// 泛型声明所声明的类型参数，运行期只能获取到占位符[K, V]
TypeVariable[] typeParameters = map.getClass().getTypeParameters();

原理

因为擦除在方法体中移除了类型信息，运行期是不存在泛型，那它是如何保证在取出数据时无需强制转型？

map.put("id", 9527);
Integer id = (Integer) map.get("id");

经研究发现，有无泛型的字节码完全一样，调用put方法由前端编译器保证，调用get方法时，指令编号27取出的数据皆为Object类型，然后再由指令编号28转型。so，泛型机制就是 在进入和离开方法的时候，由编译器在编译期执行类型检查并插入转型代码。
