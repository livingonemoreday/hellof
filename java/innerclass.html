  • 设计模式

  • 内部类自动拥有对其外部类所有成员的访问权，当某个外部类对象创建一个内部对象时， 此内部对象必定会秘密地捕获一个指向那个外围类对象的引用。然后，在访问此外围类成员时， 就是用那个引用来选择外围类的成员。
  • 生成对外部类对象的引用，可以使用外部类的名字后面紧跟圆点和this(Outer.this)
  • 告知某些其他对象，去创建其某个内部类的对象，此时，必须在new表达式中提供对其外部类对象的引用，需要使用.new语法
public class DotNew {
    public class Inner{}
    public static void main(String[] args) {
        DotNew dotNew = new DotNew();
        Inner inner = dotNew.new Inner();
    }
}

  • 在拥有外部类对象之前是不可能创建内部类对象的。因为内部类对象会暗暗地连接到创建它的外部类对象上， 但是，如果创建的是嵌套类（静态内部类），那就不需要对外部类对象的引用。
  • 局部内部类只能作用于其外层大括号内
  • 匿名内部类同正规的继承相比有些受限，因为匿名内部类既可以扩展类，也可以实现接口，但是不能两者兼备， 而且如果是实现接口，也只能实现一个接口
  • 嵌套类（static内部类）：创建嵌套类，并不需要外围类的对象；不能从嵌套类的对象中访问非静态的外围类对象； 普通内部类的字段和方法，只能放在类的外部层次上，所以普通的内部类不能有static数据和static字段， 也不能包含嵌套类。but 嵌套类却可以 --外部类编译时 未为普通内部类分配空间，所以就不能为内部类的静态成员分配空间，所以普通内部类不能有static成员
  • 每个内部类都能独立地继承自一个（接口的）实现，所以无论外围类是否已经继承了某个（接口的）实现，对于内部类都没有影响 内部类有效地实现了“多重继承”
