git/tree/commit对象

  • echo 'hello world' | git hash-object [-w(写入git库.git/object/)] --stdin
  • git hash-object -w filename
  • git cat-file -p|-t hashvalue
  • git update-index [--add(文件首次置于暂存区)] -cacheinfo 100644 git对象的哈希值 filename #置于暂存区
  • git update-index [--add] filename #直接构建git对象并且置于暂存区
  • git ls-files -s #查看暂存区
  • git write-tree #创建树对象
  • git commit-tree tree对象哈希值 -m 'commit message' #树对象创建提交对象 更新本地库

存储

  • git stash #将未完成的修改（就是没有提交的东西）保存到一个栈上，然后该分支会呈现出clean的状态，自然当前修改的东西也不在工作区
  • git stash apply [stash@{No.}应用具体的哪个存储，不指定的话应用最近的] #重新应用这些改动
  • git stash list #查看有哪些存储于栈上
  • git stash pop #应用最近的存储并将其从栈上扔掉
  • git stash drop [stash@{No.}不指定就移除最近的一个]

  • git commit --amend [-m 'message'] #重新提交，git log就不会显示上一次的提交。如果暂存区没有变动，就只会重新重新创建提交对象，这种情况实则只是修改了commit的message，如果暂存区有变动，则会创建tree对象和提交对象，这种情况就是本该一次提交的东西没有修改完全或者修改没有全部纳入暂存区作为一次提交
  • git reset --hard|mixed|soft hashvalue

标签

  • git tag [-l 'tagName'] #列出[tagName之后的]标签
  • git tag tagName [commitHash] [-m 'message'] #于commitHash(默认HEAD)处创建标签
  • git show tagName|hashvalue
  • git push origin tagName|--tags(所有不在远程仓库的本地标签) #git push并不会推送标签
  • git tag -d tagName #删除本地仓库标签
  • git push origin :refs/tags/tagName #删除本地标签后更新远程标签
