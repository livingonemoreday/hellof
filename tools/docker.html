安装(ce on centos7)

https://docs.docker.com/install/linux/docker-ce/centos/ (opens new window)

Uninstall old versions

Older versions of Docker were called docker or docker-engine. If these are installed, uninstall them, along with associated dependencies.

sudo yum remove docker \
              docker-client \
              docker-client-latest \
              docker-common \
              docker-latest \
              docker-latest-logrotate \
              docker-logrotate \
              docker-engine

Install using the repository

Before you install Docker Engine - Community for the first time on a new host machine, you need to set up the Docker repository. Afterward, you can install and update Docker from the repository.

Set up the repository

  • sudo yum install -y yum-utils device-mapper-persistent-data lvm2
  • sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
  • sudo yum-config-manager --enable docker-ce-nightly
  • sudo yum-config-manager --enable docker-ce-test
  • sudo yum-config-manager --disable docker-ce-nightly

Install Docker Engine - Community

  • sudo yum install docker-ce docker-ce-cli containerd.io
  • sudo systemctl start docker
  • systemctl status docker

常用命令

  • docker version/info/--help
  • docker images
    • -a: 列出本地所有镜像包括中间层
    • -q: 只显示镜像ID
    • --digests: 显示镜像摘要信息
    • --no-trunc: 显示完整镜像信息
  • docker images imageName
  • docker pull imageName[:tag]
  • docker rmi [-f] imageID #多个则空格分隔 删除镜像
  • docker rmi [-f] $(docker images -qa)
  • docker run -it -v 宿主机绝对路径:容器内绝对路径[:ro] imageName #ro read only不能在容器中修改，可以在宿主机上修改共享给容器 -p端口映射 -P随机端口
  • docker run -it --name dc02 --volumes-from dc01 imageName #容器间数据传递共享，实例容器dc02继承dc01
  • exit #容器停止，退出
  • crtl + p + q #容器不停止推出
  • docker start/restart/stop/kill/rm containerID或容器名
  • docker rm -f $(docker ps -a -q) #删除容器
  • docker ps -a -q | xargs docker rm
  • docker logs -f -t --tail containerID #t 时间戳
  • docker top containerID #查看该容器内运行的进程
  • docker inspect containerID #查看容器内部细节，比如数据卷 volumes
  • docker exec -it containerID /bin/bash #进入正在运行的容器并以命令行交互，这个会启动新的进程
  • docker exec -it containerID ls -l #在不进去容器的情况下
  • docker attach containerID #重新进入容器，不会启动新的进程，可进入容器后ps -ef对比上一个命令
  • docker cp containerID:容器内路径 目的主机路径 #从容器内拷贝文件到主机
  • docker commit -m='提交描述信息' -a='作者' containerID 目标镜像名称 #根据容器生成镜像

Dockerfile

  • FROM #
  • MAINTAINER #姓名和邮箱地址
  • RUN #linux 命令
  • EXPOSE #当前容器对外端口
  • WORKDIR #登录进容器后的目录
  • ENV #环境变量
  • COPY #
  • ADD #将宿主机目录中的文件拷贝到容器且自动处理URL和解压tar压缩包
  • VOLUME #数据卷， 宿主机默认是/var/lib/docker/volumes/hash值/_data/
  • CMD #容器启动命令
  • ENTRYPOINT #容器启动命令， 不同于CMD之处在于docker run最后的参数可以传递给entrypoint后的启动命令，而覆盖cmd后的启动命令
  • ONBUILD #子镜像FROM父镜像，在build子镜像的时候，如果父镜像里面有ONBUILD（即父镜像的dockerfile里面有ONBUILD），会触发ONBUILD后面的命令
