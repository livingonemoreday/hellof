Nginx是服务端负载均衡，客户端所有请求都会交给nginx，然后由nginx实现转发请求。

Ribbon是客户端负载均衡，在调用微服务接口时，从注册中心上获取注册信息服务列表，而后根据均衡策略选择适宜的服务地址调用。

http://docs.springcloud.cn/user-guide/ribbon/#ribbon_1 (opens new window)
默认轮询策略，若修改需将配置目标策略（IRule的实现类）的配置类置于application的包扫描外（以下示例中则是SpecialRibbonRule该类不可在application的包扫描路径中）， 如果置于包扫描路径中，则不能结合@RibbonClient针对特定的服务使用特定的策略，而是对于所有服务都是那个策略了。

@Configuration
public class SpecialRibbonRule {

    @Bean
    public IRule specialRule() {
        return new RandomRule();
    }
}
