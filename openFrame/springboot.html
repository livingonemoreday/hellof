配置

ConfigurationProperties

默认从全局配置文件application.yml或application.properties取值

// 将配置文件中的属性映射到该组件 该注解生效的前提：需要将改组件加入到容器中
@ConfigurationProperties(prefix = "person")
@Component
// 该bean注入的属性需要校验
@Validated
public class Person {

    // 表示 校验该属性值必须为@Email格式
    // @Email
    private String name;
    private Integer age;
    private Map<Object, Object> map;
    private List list;
    // 省略getter,setter,toString等
}

PropertySource

从指定的配置文件中取值

// 从类路径下的cat.properties文件中取值
@PropertySource(value = {"classpath:cat.properties"})
@Component
@ConfigurationProperties(prefix = "cat")
public class Cat {

    private String name;
    private Integer age;
    // 省略getter,setter,toString等
}

Application property files

SpringApplication will load properties from application.properties files in the following locations and add them to the Spring Environment:

  • A /config subdirectory of the current directory.
  • The current directory
  • A classpath /config package
  • The classpath root

权重从高到地，且互补配置

You can also refer to an explicit location using the spring.config.location environment property (comma-separated list of directory locations, or file paths).
java -jar myproject.jar --spring.config.location=E:/xx
