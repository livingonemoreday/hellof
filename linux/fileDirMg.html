目录与路径

  • -: 代表前一个工作目录
  • ~account: 代表account这个使用者的家目录
  • mkdir [-mp] 目录名称
    • -m: 直接设置权限，不使用默认权限umask
    • -p: 可递归创建

文件目录管理

ls

cp

  • cp [-adfilprsu] source destination
    • -a: 相当于-dr --preserve=all
    • -i: 若目标文件destination已存在，则在覆盖时会先询问操作的进行
    • -p: 连同文件的属性（权限/用户/时间）一起复制过去
    • -r: 递归，用于目录的复制操作
    • -d: 若source为link file，则复制link file，而非文件本身
    • -u: destination比source旧才更新，或destination不存在才复制
    • --preserve=all: 除了-p的权限参数外，还有SELinux属性
    • 若源文件source不止一个，则destination必须时目录

rm

  • rm [-fir] 文件或目录
    • -i: 交互模式，删除前总会询问

mv

  • mv [-fiu] source destination
    • -u: 若目标文件已存在，且source较新，才会覆盖

文件内容查看

cat

  • cat [-AbEnTv] filename
    • -A: 等同于-vET
    • -E: 将结尾的换行符$显示出来
    • -T: 将[Tab]按键以^T显示出来，windows的换行符为^M$
    • -v: 列出一些看不出来的特殊字符
    • -n: 打印行号，连同空白行也会有行号
    • -b: 打印行号，仅针对非空白行
  • tac反向显示

more

less

  • /hello: 向下查找字符串hello
  • ?hello: 向上查找字符串hello
  • n: 重复前一个查找
  • N: 反向重复前一个查找
  • g/G: 至首/尾行

head

  • head -n 10 filename: 显示前10行
  • head -n -10 filename: 后面10行不显示

tail

  • tail -f filename: 监听
  • tail -n 10 filename: 显示最后10行
  • tail -n +10 filename: 显示从10行开始的数据

od(非纯文本)

  • od [-t TYPE] filename，TYPE取值
    • c: 使用ASCII字符来输出

touch

  • 文件时间参数,默认ls显示的时mtime
    • 修改时间（mtime)
    • 状态时间（ctime)
    • 读取时间（atime)
  • touch [-acdmt] filename
    • -a: 仅自定义atime
    • -c：仅修改文件时间，若文件不存在则不建立新文件
    • -d: -d "2 days ago"
    • -t: -t 20191111
    • -m: 仅修改mtime

umask

用于用户在建立文件或目录时的权限默认值

  • umask [-S]

隐藏属性

  • chattr [+-=] [ASacdistu] filename
    • -/+/=: 删除/新增/赋予特殊参数
    • a: 当设置a后，该文件只能增加数据，而不能删除也不能修改数据，仅有root用户才能设置该属性
    • i: 让文件不能被删除/改名/设置链接/写入/新增数据，仅有root方能设置
  • lsattr [-adR] filename

特殊权限

SUID(Set UID 4)

s出现在文件拥有者的x权限上时，eg:
[yu@103 ~]$ ls -ld /usr/bin/passwd
-rwsr-xr-x. 1 root root 27832 6月 10 2014 /usr/bin/passwd

  • SUID权限仅对二进制程序有效，不能够用在shell脚本上
  • 执行者对于该程序需要具有x权限
  • 本权限仅在执行该程序的过程中有效
  • 执行者将具有该程序拥有者的权限

SGID(Set GID 2)

s出现在拥有者的x权限上，对文件目录皆可进行设置

SBIT(Sticky Bit 1)

只针对目录，t出现在其他组人的x权限上，用户在该目录下建立的文件或目录，只有自己和root才有权力删除

file观察文件类型

查找

which

根据PATH环境变量规范的路径查找

whereis

之查找系统中某些特定目录下的文件，eg:/bin/sbin;/usr/share/man

locate/updatedb

由数据库/var/lib/mlocate里面的数据查找，该数据库默认每天执行一次更新，可用updatedb手动更新

  • locate [-ir] keyword
    • -r: 后面可接正则表达式的显示方式
    • -l: 仅输出几行，-l 5

find

find [PATH] [option] [action]

  • 时间相关的选项:-atime/ctime/mtime
    • -mtime n: 在n天前的那一天内被修改过内容的文件
    • -mtime +n: 在n天之前被修改过内容的文件
    • -mtime -n: 在n天之内被修改过内容的文件
    • -newer file: 比file还要新的文件
  • 使用者或用户组名称相关
    • -uid n:
    • -gid n:
    • -user name:
    • -group name:
    • -nouser: 文件拥有者不在/etc/passwd中的
    • -nogroup:
  • 文件权限及名称有关
    • -name filename
    • -size [+-] nc/k: 为数字，c为Byte，k为kB
    • -type Type: f/b/c/d/l/s/p
    • -perm mode: 文件权限刚好为mode
    • -perm -mode: 文件权限全部囊括mode
    • -perm /mode: 文件权限包含任一mode的权限
  • 额外进行的操作
    • -exec command:
      [yu@103 ~]$ find /usr/bin /usr/sbin -perm /7000 -exec ls -l {} \;  
  # {}代表find找到的内容
  # -exec ;是关键词，因;有特殊意义，所以\转义
